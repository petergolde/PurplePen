File/Create RouteGadget Files...

This command creates files for use in the RouteGadget route display program. RouteGadget is a web-based program that allows competitors to draw their routes on a copy of the map and compare them with other competitors. You can learn more about RouteGadget here.

RouteGadget requires uploading three or four files to the web site for an event:

  • A .gif file with an image of the map
  • If your map is georeferenced, a .gfw file with georeferencing information
  • A .xml file that describes the locations of the controls and the courses
  • A file from the timing program with split times for each competitor

This command automatically creates the first two (three if georeferenced) files. After the event is complete, you must obtain the timing program output before posting the event to Route Gadget.

When creating the image of the map, Purple Pen automatically creates a map image of the appropriate resolution that encloses all the print areas.

 

This dialog has several options to customize the result.

File Name section

This section allows setting the file name (without the .xml or .gif extension) for the two files that are created. The default name is the name of the event.

Folder section

This section determines in which folder (directory) the RouteGadget files are created. There are three choices:

  • Same folder as Purple Pen file. The files are placed in the same folder as the Purple Pen (.ppen) file for this event. This is the default choice.
  • Same folder as map file. The files are placed in the same folder as the map file for this event.
  • Other folder. Any other folder. If you chose this, a "Select Folder..." button appears which you can click to select any folder.

IOF XML section

This sections allows you specify which version of IOF XML to use. The default (3.0), should work for almost all cases. Very old RouteGadget installations may require 2.0.3.

Other buttons

Create: This button Creates the RouteGadget files. The dialog box closes.

Cancel: Closes the dialog box without creating anything.
